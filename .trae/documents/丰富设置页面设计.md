# 丰富设置页面设计计划

## Summary
对 `app/(tabs)/settings.tsx` 进行全面视觉升级，添加数据概览头部，重新设计所有功能分组，打造更现代、信息更丰富的设置体验。

## Current State Analysis
当前设置页面为纯列表式布局，存在以下问题：
1. **视觉层次单一** - 所有行元素存在感相同，缺乏主次之分
2. **无数据预览** - 设置页作为高频入口，未展示任何衣橱核心数据
3. **Section 视觉弱** - 分组标签仅为小字，与内容粘连
4. **交互反馈简陋** - 仅有基础按压变色，缺少更细腻的动效
5. **缺少个性化/偏好设置** - 无开关类控件，功能全为跳转或输入

## Proposed Changes

### 文件：`app/(tabs)/settings.tsx`

#### 1. 添加数据概览头部（Profile + Stats）
- **Profile 卡片**：显示应用图标（大图标或 Logo 区域）、应用名称 "Wardrobe"、版本号、衣物总数
  - 样式：使用 `Colors.surfaceElevated` 背景，圆角 `Radius.xl`，左边框 3px 使用 `Colors.accent`
  - 数据：从 `useClothingStore` 取 `items.length`
- **3 个关键数据小卡片**（横向排列）：
  - 总件数 `items.length`
  - 总价值 `items.reduce((sum, i) => sum + parsePrice(i.price), 0)`
  - 品牌数 `new Set(items.map(i => i.brand).filter(Boolean)).size`
  - 样式：`Colors.surfaceElevated` 背景，圆角 `Radius.lg`，数字用 `Colors.accent` 大字号，标签用 `Colors.textSecondary`
  - 点击整个 Profile 区域可跳转 `/settings/statistics`

#### 2. 重新设计 Section 分组
所有 Section 统一使用新的视觉语言：
- **Section Header**：取消原来的小字标签，改为卡片内嵌标题行（带图标+标题+可选副标题），与下方内容一体化
- **Card 样式**：保持 `Colors.surfaceElevated` + `Radius.xl` + `Colors.border`，但取消 `overflow: 'hidden'` 以便支持阴影（如需要）
- **Row 组件升级**：
  - 支持 `value` 属性（右侧显示状态文字，如 "已开启"、"v1.0"）
  - 支持 `rightElement` 属性（可传入 Switch 等自定义元素）
  - 图标背景使用分类色彩：数据-蓝、导入-绿、同步-紫、统计-琥珀、管理-青、偏好-粉、危险-红
  - 按压效果从纯色变色改为 `scale(0.98)` + 背景高亮（使用 `Animated` 或 `Pressable` 的 `pressed` 状态）

#### 3. Section 重组与新增
按以下顺序重组：

| Section | 内容 | 说明 |
|---------|------|------|
| 数据管理 | 导出备份、恢复备份 | 保持原有功能 |
| 导入 | 从备份文件导入 | 保持原有功能，样式升级为标准 Row |
| 同步设置 | 服务器地址、Token | 保持原有输入框，但放入统一 Card |
| 统计与分析 | 统计信息（跳转）| 右侧显示 "查看" + 箭头 |
| 管理 | 分类管理、回收站 | 保持原有跳转 |
| 偏好设置（新增）| 显示价格（Switch）、每日提醒（Switch）| 使用 `react-native` 的 `Switch` 组件，颜色绑定 `Colors.accent`；先仅做 UI 状态，数据持久化可后续补充 |
| 关于（新增）| 版本 v1.0、开源许可 | 静态信息行 |
| 危险操作 | 清空衣橱 | 保持原有功能，红色图标背景 |

#### 4. 输入框样式优化
同步设置中的输入框：
- 背景改为 `Colors.surface`（比卡片稍暗，形成层级）
- 左侧添加小图标（`link-outline`、`key-outline`）
- 聚焦时边框变为 `Colors.accentMuted`

#### 5. Footer 升级
- 版本信息移入"关于" Section
- 底部仅保留极简版权文字 `"© 2025 Wardrobe"`

#### 6. 代码结构优化
- 提取 `SettingRow` 为独立子组件（文件内），接收 `icon`、`title`、`subtitle?`、`value?`、`rightElement?`、`danger?`、`onPress?`、`iconColor?`、`iconBg?`
- 提取 `StatCard` 子组件（文件内），接收 `value`、`label`、`onPress?`

## Assumptions & Decisions
- **偏好设置开关**：仅做本地 `useState` 状态，不接入 `database.ts` 的 setting 表，避免过度扩展；如需持久化可后续在 `database.ts` 的 `getSetting`/`setSetting` 基础上扩展。
- **价格解析**：复用 `statistics.tsx` 中的 `parsePrice` 逻辑（提取到 utils 或内联复制），避免引入新依赖。
- **色彩分配**：Section 图标背景色使用设计令牌中已有的功能色/分类色，不新增颜色：
  - 数据：`Colors.catBag` (#8FC8D4)
  - 导入：`Colors.success` (#6B9B6F)
  - 同步：`Colors.catOuter` (#B8A0D9)
  - 统计：`Colors.accent` (#E8B86D)
  - 管理：`Colors.catShoes` (#8FD4A8)
  - 偏好：`Colors.catDress` (#D48FB3)
  - 关于：`Colors.textSecondary` (#9CA3AF)
  - 危险：`Colors.danger` (#E07A5F)
- **动画**：不使用 `Animated` API（避免过度工程），仅使用 `Pressable` 的 `pressed` 状态做 `transform: [{ scale: 0.98 }]` 和背景色变化。

## Verification Steps
1. 启动应用并进入设置 Tab，确认顶部出现 Profile 卡片和 3 个数据小卡片
2. 点击数据卡片或 Profile 卡片，确认可跳转统计页面
3. 确认所有 Section 分组清晰，图标色彩各异
4. 确认偏好设置中的 Switch 可以正常切换状态
5. 确认输入框聚焦时边框变色
6. 确认危险操作行仍为红色
7. 确认各按压交互有 scale 反馈
8. 在 iOS/Android/Web 三端检查布局无错位
