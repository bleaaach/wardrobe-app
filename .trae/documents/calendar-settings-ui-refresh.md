# 日历与设置页面 UI 优化计划

## Summary

对 `calendar.tsx` 和 `settings.tsx` 进行全面 UI 重构。核心思路：**删减元素、强化视觉层次、统一设计语言、提升信息密度与呼吸感的平衡**。保持与现有设计系统（tokens.ts）和首页杂志风的一致性。

---

## Current State Analysis

### 日历页面 (app/(tabs)/calendar.tsx)

**问题 1: 头部信息冗余，抢夺主视觉**
- 42px 的 "Calendar" 英文大标题 + 副标题占据顶部 60+px 空间，但用户打开此页面的目的是看日历，不是看标题。
- **违反原则**：主役不明确，标题喧宾夺主。

**问题 2: 日历网格视觉噪声严重**
- 日期数字、照片缩略图、选中底色、今天边框、照片上的深色遮罩标签，5 层信息在同一个 DAY_SIZE 格子里堆叠。
- 有照片的日期和无照片的日期视觉权重差异过大，导致日历看起来像"打补丁"。
- **违反原则**：70分元素均一排列，没有120分的焦点。

**问题 3: 详情卡片 (detailCard) 内容堆砌**
- 穿搭预览(280px) + 穿搭信息 + 物品横向列表 + 操作按钮，全部塞在一个卡片里，缺乏分段和呼吸感。
- 空状态时的 4 个选项卡片（从搭配库/从物品库/从相册/复制昨日）全部平铺，没有操作优先级。

**问题 4: 颜色硬编码与主题脱节**
- 选项卡片的 icon 背景色使用硬编码 `rgba(232,184,109,0.15)` 等，与主题系统的分类色（catBag, catDress 等）混用，风格不统一。

---

### 设置页面 (app/(tabs)/settings.tsx)

**问题 1: 信息过载，Section 过多**
- 9 个 Section（Profile、数据管理、导入、同步设置、WebDAV、统计、管理、偏好、关于、危险操作），用户滚动时找不到重点。
- **违反原则**：1画面1动作的理念被完全打破。

**问题 2: 输入框直接暴露在列表中**
- 服务器地址、Token、WebDAV 地址/用户名/密码等输入框直接放在 SectionCard 里，看起来像未完成的表单，与设置项的 Row 风格不一致。
- 输入框没有统一的容器包裹，聚焦时只有 borderColor 变化，视觉反馈弱。

**问题 3: 图标颜色过于花哨**
- 每个 SettingRow 使用不同的 iconBg/iconColor（蓝、绿、粉、橙、灰），像彩虹，缺乏整体感。
- **违反原则**：视觉噪声过多，没有统一的辅助色层级。

**问题 4: 统计卡片缺乏设计层次**
- 3 个 StatCard 平铺，数字和标签的排版过于基础，没有数据感。

**问题 5: 操作层级混乱**
- WebDAV 区块中，"测试连接"、"立即同步"、"从 WebDAV 恢复" 重要性差异巨大，却使用相同的 Row 样式并列展示。

## Proposed Changes

### 日历页面 (app/(tabs)/calendar.tsx)

#### Change 1: 重构头部 — 极简月份导航
**What**: 删除 "Calendar" 大标题和副标题，将年份和月份导航合并为单行紧凑布局。
**Why**: 页面主役是日历网格，不是标题。减少顶部空间浪费。
**How**:
- 删除 `header` 和 `headerTitle` / `headerSub` 样式。
- 在 `monthCard` 顶部放置单行导航：`‹ 2025年 ›` 和 `‹ 6月 ›` 左右并排，或合并为 `‹  2025年 6月  ›`。
- 月份文字加大为 `FontSize.xxl`，作为页面唯一的主视觉锚点。

#### Change 2: 日历网格 — 照片与日期分离
**What**: 有穿搭记录的日期不再在格子内显示照片缩略图，改为显示一个精致的"已记录"指示点（或小型徽章）。点击后在详情区展示大图。
**Why**: 日历网格的职责是让用户快速定位"哪天有记录"，不是展示穿搭细节。照片缩略图在 DAY_SIZE（约40-50px）里几乎看不清，反而造成视觉混乱。
**How**:
- 删除 `dayPhoto`、`dayPhotoSel`、`dayLabelOnPhoto`、`dayTextOnPhoto` 相关的绝对定位和遮罩逻辑。
- 有记录的日期：在数字下方显示 4px 的 accent 色圆点（或日期数字本身用 accent 色）。
- 选中的日期：保持 `daySel` 的 accent 背景色，但简化动画。
- 今天的日期：从"边框"改为"文字加粗 + accent 色 + 下方小点"，减少边框带来的盒子感。

#### Change 3: 详情卡片 — 分段式布局
**What**: 将 detailCard 拆分为三个视觉层次：日期头、内容区、操作区。
**Why**: 通过分段让用户一眼识别"这是什么"、"内容是什么"、"我能做什么"。
**How**:
- 日期头：保留 `detailDate` 和 `todayBadge`，但加大日期文字的视觉权重。
- 内容区：穿搭照片/OutfitPreview 放大到撑满卡片宽度（marginHorizontal 取消），采用全宽展示（类似首页 hero 风格）。
- 物品列表：从横向小卡片改为纵向紧凑列表（或保持横向但减少卡片尺寸和边框）。
- 操作区："更换穿搭"和"删除"按钮改为底部固定样式，Primary 和 Danger 的区分更明显。

#### Change 4: 空状态与选项 — 主次分明的操作面板
**What**: 4 个选项改为 2+2 的网格，主操作（从搭配库、从相册）在前，次操作（从物品库、复制昨日）在后。
**Why**: 引导用户优先使用"搭配库"（产品核心功能）和"相册"（最自然记录方式）。
**How**:
- 统一 4 个选项的 icon 背景色为 `colors.surfaceHighlight`，只保留 icon 本身使用 accent 或分类色。
- 选项卡片改为更紧凑的 pill 形按钮或简洁的 icon+文字组合，减少边框和背景色差异。

### 设置页面 (app/(tabs)/settings.tsx)

#### Change 1: 合并与精简 Section
**What**: 将 9 个 Section 合并为 4 个核心区块：账户概览、数据与同步、偏好设置、关于与危险操作。
**Why**: 大幅减少用户认知负担，让高频设置项更容易触达。
**How**:
- **账户概览**: ProfileCard + 3 个 StatCard（保留，但优化样式）。
- **数据与同步**: 合并"数据管理"、"导入"、"同步设置"、"WebDAV"为一个可展开的区块。默认只显示"导出备份"、"恢复备份"、"WebDAV 同步"三个入口，点击后展开详细表单。
- **偏好设置**: 深色模式、显示价格、每日提醒（保留）。
- **关于与危险操作**: 版本、开源许可、清空衣橱（底部放置，用分割线隔开）。

#### Change 2: 输入框表单化 — 统一的 InputGroup 容器
**What**: 服务器地址、Token、WebDAV 配置等输入框不再直接放在 SectionCard 里，而是使用统一的表单容器包裹。
**Why**: 提升输入框的视觉完整性和聚焦反馈。
**How**:
- 新增 `inputGroup` 样式：灰色/浅色背景（`colors.surface`）包裹，圆角 `Radius.lg`，内部输入框去掉独立边框，使用底部细线分隔（仅最后一项无底线）。
- 聚焦时整个 Group 不变，只有当前输入框的 label 变为 accent 色。
- 表单默认折叠，点击"配置服务器"或"配置 WebDAV"后才展开。

#### Change 3: 统一图标配色体系
**What**: 所有 SettingRow 的 icon 背景统一为 `colors.surfaceHighlight`，图标颜色只使用 3 种：accent（主要）、textSecondary（中性）、danger（危险）。
**Why**: 消除彩虹效应，建立清晰的视觉语义。
**How**:
- 删除所有硬编码的 `iconBg` 和 `iconColor`（除了 danger）。
- 默认使用 `colors.accentLight` 背景 + `colors.accent` 图标。
- 危险操作（清空衣橱）使用 `colors.danger` + `rgba(224,122,95,0.15)`。
- 中性信息（版本、开源许可）使用 `colors.surfaceHighlight` 背景 + `colors.textSecondary` 图标。

#### Change 4: 统计卡片数据化设计
**What**: StatCard 改为更紧凑、有数据仪表盘感的样式。
**Why**: 当前 StatCard 像普通的按钮，缺乏数据展示的专业感。
**How**:
- 数字使用 `FontSize.xxl`，标签使用 `FontSize.xs` + 全大写或加宽字间距。
- 卡片背景改为渐变或左侧 accent 色条（4px）点缀，增加辨识度。
- 3 个卡片间距缩小，作为一个整体区块出现。

#### Change 5: WebDAV 操作层级重构
**What**: "测试连接"改为输入框区域的辅助按钮（小号 outline 按钮），"立即同步"和"从 WebDAV 恢复"改为 Primary 和 Danger 样式的独立大按钮。
**Why**: 区分"配置验证"和"数据操作"的风险等级。
**How**:
- 测试连接：放在输入框下方，小号文字按钮（`text-only` 或 `outline` 样式）。
- 立即同步：使用 `actionBtnPrimary` 样式，全宽按钮。
- 从 WebDAV 恢复：折叠在"高级"菜单下，避免误触。

## Assumptions & Decisions

1. **保持 React Native 技术栈**：不引入新依赖，不使用 Web 专属的 CSS（如 backdrop-filter）。所有改动基于现有 `StyleSheet` 和 `design/tokens`。
2. **暗色/亮色双主题兼容**：所有颜色引用必须使用 `colors.xxx`，不使用硬编码色值（除了已存在于 tokens 中的语义化 rgba）。
3. **不改动业务逻辑**：仅调整布局、样式和组件结构，保留所有 state、handler、navigation 逻辑。
4. **杂志风一致性**：参考首页 (`index.tsx`) 的 hero overlay、大圆角、留白风格，让日历和设置与首页的设计语言统一。
5. **删除优先**：日历页面的照片缩略图从格子中移除是重大决策，但基于"日历=定位工具，详情=展示工具"的信息架构原则，这是正确的删减。

## Verification Steps

1. **视觉检查**：
   - [ ] 日历页面顶部不再有"Calendar"大标题，月份导航紧凑。
   - [ ] 日历格子不再有照片重叠，有记录的日期通过小点或文字颜色区分。
   - [ ] 设置页面 Section 数量从 9 个减少到 4 个。
   - [ ] 设置页面图标颜色不再花哨，统一为 accent/中性/危险三种。

2. **交互检查**：
   - [ ] 日历选中日期后，详情区正确展示该日期的穿搭/照片。
   - [ ] 设置页面的 WebDAV 输入框在折叠/展开状态下正常工作。
   - [ ] 所有 Pressable、Switch、TextInput 的交互反馈保留。

3. **主题检查**：
   - [ ] 切换浅色/深色模式后，两个页面的文字对比度正常，无看不清的元素。
   - [ ] 无硬编码颜色导致的主题切换后突兀问题。

4. **设备检查**：
   - [ ] iOS 与 Android 的日历格子宽度计算正常（DAY_SIZE 基于 SCREEN_W）。
   - [ ] 设置页面 ScrollView 底部留有足够安全区域（bottom padding）。
